## Task

Build the timestamped rolling audio timeline and best-effort mixer. Retain exactly the latest 30 seconds of both normalized audio sources in bounded memory and produce one aligned inference snapshot on demand.

## What was tried

### 1. Initial implementation

89 tests passed but Spec review failed (Spec 65, Tests 72, averaged 76.7). Missing evidence for:
- overlap/out-of-order/drift handling
- hard-clipping mixer (no headroom)
- swallowed conversion errors
- shared implicit end timestamp between segments
- O(n*m) extraction complexity

### 2. First correction

105 tests passed and Spec/Quality passed, but Security failed at 85 due to:
- `channelCount == 0` division by zero before validation
- WAV narrowing (potential truncation)
- segment fragmentation (unbounded growth)
- nonfinite samples not guarded
- unchecked `Sendable` concerns

### 3. Second correction

117 tests pass and Spec passed, but Code Quality failed. Strict command `swift build -c release -Xswiftc -strict-concurrency=complete -Xswiftc -warnings-as-errors` reports:
- captured mutable `hasProvidedInput` reference/mutation errors at `AudioPipeline.swift` callback
- `AudioConverter.convert`: 67 lines (>50 limit)
- `WAVEncoder.encodePCM16`: 51 lines (>50 limit)
- `AudioTimeline.insert`: 51 lines (>50 limit)
- Reviewer averaged 88.7

## Why it failed

Fixes accumulated without keeping strict warning/function-size gates continuously green. Each round addressed only the previous review category, but regressions in other dimensions were not caught because the full gate suite was not re-run after each change.

## Suggested next step

1. Split `AudioConverter.convert` into decode/header/overlap helpers (target ≤50 lines each).
2. Split `WAVEncoder.encodePCM16` into header-body helpers.
3. Split `AudioTimeline.insert` into validation-insertion helpers.
4. Replace mutable callback capture in `AudioPipeline.swift` with a concurrency-safe single-consumption input provider (e.g., `AsyncStream` or an `actor`-backed buffer).
5. Rerun all three reviews (Spec, Security, Code Quality) after each change to keep all gates continuously green.
