## Task

Prove the app meets all acceptance, privacy, and latency requirements via end-to-end live testing on a real Mac: 10-minute idle produces no artifacts, physical hotkey triggers capture→transcribe→reason→TTS under 20 seconds, everything is recorded and artifacts cleaned up afterward. All dimensions must cross the 90 gate.

## What was tried

### 1. First attempt

296 tests + standard build pass; strict build fails. Live screen-capture scenarios, 10-minute idle, and <20s latency all unverified. Review failed — Spec 55, Tests 70, Perf 40, Evidence 50, averaged 59.3.

### 2. Second attempt

QA found 3 prior ready launches, then ScreenCaptureKit exhaustion occurred. TCC was reset, causing permission denial. Still no 10-minute idle test, no live hotkey test, no latency measurement. Review failed — Spec 35, Security 85, averaged 52.3.

### 3. Third attempt

Current TCC denied; tests pass, strict fails, live capture/hotkey/TTS/idle/latency all unverified. Review failed — Spec 68, Security 87, averaged 66.7.

## Why it failed

1. **Task 4 strict debt** — the `AudioPipeline.swift` mutable callback capture (`hasProvidedInput`) and oversized functions prevent a green strict build, which is a prerequisite gate.
2. **Task 6/7 lifecycle debt** — `OpenCodeClient` and `PipelineRunner` have unresolved Sendable/isolation issues that surface inconsistent behavior under real runtime conditions.
3. **ScreenCaptureKit TCC reset** — debugging ScreenCaptureKit exhaustion required resetting TCC, which invalidates the signed-app permission grant. Re-approval requires a full GUI sequence that cannot be scripted.
4. **Physical hotkey dependency** — the Control-Option-Space hotkey requires Accessibility permissions and a physical keyboard; it cannot be injected or simulated in automated tests without granting Accessibility to the test runner.
5. **No end-to-end benchmark harness** — there is no automated way to measure hotkey→TTS latency, verify 10-minute idle artifact cleanup, or run multi-language/multi-scenario acceptance passes without a human physically operating the machine.

## Suggested next step

1. Re-approve TCC for ScreenCaptureKit (grant in System Settings → Privacy & Security → Screen Recording).
2. Fix strict concurrency debt from Task 4 (replace mutable callback capture in `AudioPipeline.swift` with `AsyncStream` or actor-backed buffer; split oversized functions).
3. Fix dangerous process kill in `LifecycleStateMachine` — use `terminate()` instead of `kill()` for whisper server to avoid ungraceful shutdown.
4. Build and run the signed app (not debugger-launched) for 10 minutes with no input; verify zero artifacts remain in output directory.
5. Physically press Control-Option-Space and verify pipeline completes; capture stage-level timestamps and confirm TTS starts within 20 seconds.
6. Cycle through non-sensitive scenarios: mic input, system audio, mixed mic+system, English, non-English (e.g., French, German).
7. Verify artifacts are cleaned up after each run and after app termination.
8. Capture all evidence (logs, timestamps, screenshots, artifact listing) and attach to review.
